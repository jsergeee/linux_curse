# Утилиты apt и dpkg 

В Ubuntu управление пакетами осуществляется тремя способами: с использованием утилиты `apt`, `dpkg` или `snap`. Самый удобный в использовании – `apt`. Это пакетный менеджер, который включает в себя набор утилит для управления пакетами. Он позволяет осуществлять поиск, установку и удаление пакетов, обновлять операционную систему, подключать репозитории. Рассмотрим параметры утилиты apt для управления пакетами: 

- `apt list package_name` — поиск пакета по имени; 

- `apt search package_name` — поиск пакета по имени и названию; 

- `apt show package_name` — посмотреть информацию о пакете; 

- `apt install package_name -y` — установить пакет; 

- `apt install package_name1 package_name2 -y` — установить два пакета; 

- `apt remove package_name` — удалить пакет, при этом сохранятся файлы с настройками; 

- `apt purge package_name` — полностью удалить пакет, включая конфигурационные файлы; 

- `apt upgrade` — обновить все установленные пакеты; 

- `apt update` — обновить информацию о пакетах в репозиториях, указанных в настройках. 

- ` sudo apt autoremove`  удаление пакетов

- `sudo apt install --only-upgrade package_name ` - обновление пакета до последней доступной версии, если она есть в репозиториях.

  ### Резюме

  - `apt upgrade` нужно вызывать явно, он не исполняется автоматически.
  - `apt install` может обновлять пакеты, если они уже установлены, но для полного обновления всех пакетов используйте `apt upgrade`.
  - `apt upgrade` не требуется сразу после `apt install`, если вы не хотите обновить остальные пакеты.

  

  **`dpkg`** — пакетный менеджер в Debian-подобных системах. Главное отличие от утилиты apt состоит в том, что dpkg работает только с локальными пакетами, он не умеет искать и устанавливать пакеты из репозиториев. Основные параметры утилиты dpkg: 

- `dpkg -l` — просмотр списка пакетов; 

- `dpkg -i package_name` — установить пакет или группу пакетов; 

- `sudo apt -f install`  - dpkg не может автоматом исправлять ошибки и для этого нужно устанвить через apt

- `dpkg -r package_name` — удалить пакет или группу пакетов.

## Альтернативная система snap-пакетов 

**snap** — это пакет, который, помимо готовой сборки самого приложения, включает в себя все необходимые зависимости и может работать почти в любом дистрибутиве Linux. В какой-то степени можно считать, что пакеты, установленные при помощи snap, — альтернатива самостоятельной сборке пакета. Пакет, установленный через snap, содержит все необходимые зависимости и может работать в любом окружении Linux. Snap состоит из двух частей: демона snapd и клиента для управления пакетами snap. Установка snapd производится командой apt install snapd -y. 
Варианты использования команды snap: 

- `snap search package_name` — поиск пакета; 
- `snap install package_name` — установка пакета; 
- s`nap refresh package_name` — обновление пакета; 
- `snap remove package_name` — удаление пакета; 
- `snap list` — просмотр установленных пакетов.

### Проблема с запуском Apache

Судя по предоставленным вами сообщениям, Apache не запускается из-за нескольких причин. Давайте разберемся с ними и найдем решение.

### Основные ошибки

1. **Не удалось определить полное доменное имя сервера**:
   - Сообщение: `AH00558: apache2: Could not reliably determine the server's fully qualified domain name, using 127.0.1.1.`
   - Решение: Добавьте директиву `ServerName` в файл конфигурации Apache. Например, откройте файл `/etc/apache2/apache2.conf` и добавьте следующую строку:

     ```apache
     ServerName localhost
     ```

2. **Адрес уже используется**:
   - Сообщение: `(98)Address already in use: AH00072: make_sock: could not bind to address [::]:8081`
   - Решение: Это означает, что порт `8081` уже занят другим процессом. Вам нужно определить, какой процесс использует этот порт, и остановить его. Для этого выполните команду:

     ```bash
     sudo lsof -i :8081
     ```

   - После этого вы можете остановить процесс, используя его PID:

     ```bash
     sudo kill <PID>
     ```

3. **Нет доступных сокетов для прослушивания**:
   - Сообщение: `no listening sockets available, shutting down`
   - Это связано с предыдущей ошибкой. После освобождения порта попробуйте снова запустить Apache.

4. **Невозможно открыть логи**:
   - Сообщение: `AH00015: Unable to open logs`
   - Убедитесь, что Apache имеет права на запись в директорию логов. Проверьте права доступа к директории `/var/log/apache2/`:

     ```bash
     sudo chown -R www-data:www-data /var/log/apache2/
     ```

### Перезапуск Apache

После внесения всех изменений и устранения проблем попробуйте снова запустить Apache:

```bash
sudo systemctl start apache2
```

### Проверка статуса

Чтобы убедиться, что Apache запустился правильно, проверьте его статус:

```bash
sudo systemctl status apache2
```

### Заключение

Следуя этим шагам, вы должны быть в состоянии устранить проблемы с запуском Apache. Если ошибки продолжают возникать, проверьте файлы логов Apache для получения дополнительной информации о проблемах.
